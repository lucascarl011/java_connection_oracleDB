# 🗄️ Java + Oracle Database — Conexão e Inserções via JDBC

Projeto Java que demonstra como conectar em um banco de dados **Oracle** utilizando **JDBC puro** e realizar inserções simples e em lote (batch).

---

| Tecnologia | Versão |
|---|---|
| Java | 22 |
| Oracle Database | 21c Express Edition |
| Oracle JDBC Driver (ojdbc11) | 23.4.0.24.05 |
| Maven | 3.x |

## 🏦 Configuração do Banco de Dados

### 1. Execute o script SQL para criar a tabela

Conecte no seu Oracle (via DBeaver, SQL*Plus ou outra ferramenta) e execute:

```sql
CREATE TABLE SEU_SCHEMA.CLIENTES (
    ID            NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    NOME          VARCHAR2(100) NOT NULL,
    EMAIL         VARCHAR2(150) NOT NULL UNIQUE,
    TELEFONE      VARCHAR2(20),
    DATA_CADASTRO DATE DEFAULT SYSDATE NOT NULL
);
```

> ⚠️ Substitua `SEU_SCHEMA` pelo nome do seu schema (ex: `CMA`, `SYSTEM`, etc.)

## ⚙️ Configuração do Projeto

### 1. Clone o repositório

```bash
git clone https://github.com/seu-usuario/JAVA_ORACLE.git
cd JAVA_ORACLE
```

### 2. Configure as credenciais do banco

Abra o arquivo `src/main/java/projeto/conexao/ConexaoOracle.java` e edite as constantes:

```java
private static final String URL     = "jdbc:oracle:thin:@localhost:1521/xepdb1";
private static final String USUARIO = "seu_usuario";
private static final String SENHA   = "sua_senha";
```

| Campo | Descrição | Exemplo |
|---|---|---|
| `URL` | Endereço do banco Oracle | `jdbc:oracle:thin:@localhost:1521/xepdb1` |
| `USUARIO` | Usuário do banco | `SYSTEM` |
| `SENHA` | Senha do usuário | `minhasenha` |

> ⚠️ **Nunca suba credenciais reais para o GitHub!** Use variáveis de ambiente em produção.

### 3. Verifique o `pom.xml`

O driver JDBC do Oracle já está configurado:

```xml
<dependencies>
    <dependency>
        <groupId>com.oracle.database.jdbc</groupId>
        <artifactId>ojdbc11</artifactId>
        <version>23.4.0.24.05</version>
    </dependency>
</dependencies>
```

Clique em **"Load Maven Changes"** no IntelliJ (ícone 🐘) para baixar a dependência.

## 📄 Descrição das Classes

### `ConexaoOracle.java`
Responsável por abrir e gerenciar a conexão com o banco Oracle via JDBC. Contém as configurações de URL, usuário e senha, além de um método utilitário para testar se a conexão está funcionando.

### `Cliente.java`
Classe modelo (POJO) que representa um registro da tabela `CLIENTES`. Contém os atributos `id`, `nome`, `email` e `telefone`, com seus respectivos getters e setters.

### `ClienteDAO.java`
Classe de acesso a dados (DAO — *Data Access Object*) com dois métodos principais:
- **`inserir(Cliente)`** — insere um único cliente e retorna o ID gerado pelo banco.
- **`inserirEmLote(List<Cliente>)`** — insere uma lista de clientes em uma única transação usando batch, muito mais eficiente para grandes volumes.

### `Main.java`
Ponto de entrada da aplicação. Demonstra o uso das classes acima: testa a conexão, realiza uma inserção única e uma inserção em lote.

## 📌 Observações

- O projeto usa **JDBC puro** sem frameworks como Hibernate ou Spring, ideal para aprendizado da base de conexão com banco de dados em Java.
- O **batch insert** (`inserirEmLote`) é executado em uma única transação — se qualquer inserção falhar, todas são revertidas (`rollback` automático).
- Certifique-se de conectar no **PDB correto** (`xepdb1`) e não no CDB raiz ao usar Oracle XE.
